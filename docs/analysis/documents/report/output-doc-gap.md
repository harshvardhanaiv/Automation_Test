# Doc Gap Analysis Report
==========================

## Missing Information
--------------------

1. **Description of Delivery Path Option**
	* The official documentation mentions "Delivery Path" but does not provide a clear explanation of what it is or how it works.
	* In the generated live documentation, it is described as allowing the report output to be delivered to a specific destination path on the server system.
2. **Explanation of Replace Default Option**
	* The official documentation only mentions "Replace Default" but does not explain its purpose or functionality.
	* In the generated live documentation, it is explained that this option allows the report output to replace the last working version instead of creating a new one.
3. **List of Available Formats**
	* The official documentation lists the available formats as:
		+ `rptdocument`
		+ `aiv-xlsx`
		+ `docx`
		+ `html`
		+ `pdf`
		+ `pptx`
		+ `xls`
		+ `xls_spudsoft`
		+ `xlsx`
	* The generated live documentation lists the same formats, but with a clearer explanation of what each format represents.
4. **Steps for Verifying Output Settings**
	* The official documentation does not provide any information on how to verify output settings.
	* In the generated live documentation, it is explained that after configuring output settings, clicking on the "Run" button will generate and store the output file in the specified location.

## Outdated Content
------------------

1. **Prerequisites**
	* The official documentation requires users to download a sample file from a link.
	* In the generated live documentation, it is assumed that users already have the required files uploaded to the application.
2. **Step 1: Accessing the Output Tab**
	* The official documentation instructs users to navigate to the report and click on the "Schedule" tab.
	* In the generated live documentation, it is explained that users should navigate to the report and click on the "Output" tab.

## Inaccuracies
----------------

1. **Step 2: Configuring Output Settings**
	* The official documentation has a typo in the label for the "Format" field (missing an "e").
	* In the generated live documentation, this typo is corrected.
2. **Field Labels**
	* The official documentation uses inconsistent field labels throughout the document (e.g., "**Field**" vs. no bolding).
	* In the generated live documentation, consistent field labeling is used.

## Recommendations
------------------

1. **Update Prerequisites Section**
	* Update the prerequisites section to reflect that users should have already uploaded the required files.
2. **Reorganize Step 1: Accessing the Output Tab**
	* Reorganize the instructions for accessing the output tab to match the steps in the generated live documentation.
3. **Add Explanation of Delivery Path Option and Replace Default Option**
	* Add explanations for these options to the official documentation.
4. **Update List of Available Formats**
	* Update the list of available formats to reflect any changes or additions made to the application.
5. **Remove Outdated Content**
	* Remove outdated content, such as steps for verifying output settings, which is not relevant in the current version of the application.
