A **scheduled report** is a report that runs automatically at a specified time or interval and can trigger specific actions each time it executes.

This document explains how to schedule reports **based on time parameters** according to your requirements.

##  <h4> Scheduling Reports by Time in the Application </h4> 

Users can schedule reports at different times based on their needs using **three scheduling options**:

---

##  <h4> 1. Right Now </h4> 

Use this option to **execute the report immediately** after scheduling.

<a href=".././rightnow" target="_blank">Click here</a> to learn more about scheduling reports to run **Right Now**.

---

##  <h4> 2. Once </h4> 

Use this option to **run the report once** at a specific date and time defined by the user.

<a href=".././once" target="_blank"> Click here </a> to learn more about scheduling reports to run **Once**.

---

##  <h4> 3. Recurring </h4> 

Use this option to **schedule the report to run repeatedly** at regular intervals — hourly, daily, weekly, monthly, or yearly.

<a href=".././recurring" target="_blank"> Click here </a> to learn more about scheduling reports to run **Recurring**.
